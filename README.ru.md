# MANTROVA STUDIO — готовый MVP-магазин

Внутри лежит полностью готовая стартовая версия:
- `site/index.html` — основной сайт
- `site/admin/index.html` — панель управления
- `site/call-center/index.html` — колл-центр
- `site/assets/style.css` — общие стили
- `site/assets/app.js` — вся фронтенд-логика
- `worker/src/index.js` — API для заказов, админки и колл-центра
- `database/schema.sql` — база D1 с начальными данными
- `worker/wrangler.toml` — конфиг Cloudflare Worker

## Важное честно

Эта версия — **готовый рабочий MVP**, который можно развернуть и использовать как основу сразу.
Но есть 3 вещи, которые вам потом, скорее всего, захочется улучшить:

1. подключить реальное геокодирование Яндекс.Карт, чтобы автоматически получать `lat/lng` из адреса;
2. загрузку файлов не по URL, а через Cloudflare R2;
3. защиту `admin` и `call-center` через Cloudflare Access поверх текущего пароля.

# КАК УСТАНОВИТЬ ПО ШАГАМ

## 1. Создайте приватный репозиторий на GitHub
Название любое, например:
`mantrova-studio-shop`

## 2. Загрузите в него файлы
Скопируйте содержимое папок так:

- всё из папки `site/` — в корень сайта Pages
- папку `worker/` оставьте как есть
- папку `database/` оставьте как есть

Проще всего держать всё в одном репозитории именно в такой структуре.

## 3. Установите Wrangler
На ПК должен быть Node.js.
Потом в терминале:

```bash
npm install -g wrangler
wrangler login
```

## 4. Создайте D1 базу
В терминале из папки `worker/`:

```bash
wrangler d1 create mantrova_studio
```

Cloudflare покажет `database_id`.

Откройте `worker/wrangler.toml` и вставьте этот ID сюда:

```toml
database_id = "PASTE_YOUR_D1_DATABASE_ID_HERE"
```

## 5. Залейте схему базы
Из папки `worker/` выполните:

```bash
wrangler d1 execute mantrova_studio --file=../database/schema.sql --remote
```

## 6. Добавьте секреты
Из папки `worker/` по очереди:

```bash
wrangler secret put ADMIN_PASSWORD
wrangler secret put TG_BOT_TOKEN
wrangler secret put TG_CHAT_ID
```

### Что вводить
- `ADMIN_PASSWORD` — пароль для входа в админку
- `TG_BOT_TOKEN` — токен Telegram-бота
- `TG_CHAT_ID` — ID чата, куда слать заказы

Если Telegram пока не нужен, токен и чат можно не добавлять.

## 7. Разверните API Worker
Из папки `worker/`:

```bash
wrangler deploy
```

После деплоя будет адрес вида:

```text
https://mantrova-studio-api.ваш-subdomain.workers.dev
```

Запомните его.

# ПОДКЛЮЧЕНИЕ FRONTEND К API

## 8. В файле `site/assets/app.js`
В самом верху есть строка:

```js
const API_BASE = window.API_BASE || '/api';
```

### Вариант 1 — самый простой
Если вы прокинете через Cloudflare route `/api/*` на worker, ничего менять не надо.

### Вариант 2 — быстро проверить работу
Замените строку на:

```js
const API_BASE = 'https://ВАШ-WORKER-URL.workers.dev/api';
```

Например:

```js
const API_BASE = 'https://mantrova-studio-api.xxx.workers.dev/api';
```

Тогда сайт сразу начнёт работать с API.

# ПУБЛИКАЦИЯ САЙТА ЧЕРЕЗ CLOUDFLARE PAGES

## 9. Создайте проект Cloudflare Pages
В Cloudflare:
- Workers & Pages
- Create application
- Pages
- Connect to Git
- подключите ваш GitHub-репозиторий

### Build settings:
- Framework preset: `None`
- Build command: оставить пустым
- Build output directory: `site`

После этого Cloudflare будет публиковать ваш сайт из папки `site`.

# ДОМЕНЫ

## 10. Подключите домены
В Cloudflare DNS:

### основной сайт
- `м-студия.рф` → Pages project

### админка
- `админ.м-студия.рф` → тот же Pages project

### колл-центр
- `колл-центр.м-студия.рф` → тот же Pages project

Так как все 3 страницы уже лежат в проекте:
- главная: `/`
- админка: `/admin/`
- колл-центр: `/call-center/`

### Если хотите именно отдельные поддомены
Тогда настройте Redirect / Rewrite:
- `админ.м-студия.рф` → `/admin/`
- `колл-центр.м-студия.рф` → `/call-center/`

Самый простой способ — через Cloudflare Bulk Redirects или Rules.

Открываете:

```text
https://ваш-домен/admin/
```

Вводите пароль `ADMIN_PASSWORD`.

---

# КАК ЗАЙТИ В КОЛЛ-ЦЕНТР

Открываете:

```text
https://ваш-домен/call-center/
```

Важно: колл-центр использует тот же admin token, который сохраняется после входа в админку.
На первом этапе это удобно.
Потом можно отдельно закрыть и админку, и колл-центр через Cloudflare Access.

---

# КАК ДОБАВЛЯТЬ ТОВАРЫ

Зайдите в админку → `Товары` → `Добавить товар`

Заполняете:
- название
- slug
- категорию
- цену
- описание
- URL картинки
- бейджи
- активность
- варианты JSON

Пример `Варианты JSON`:

```json
[
  { "name": "A4 без рамки", "price": 1900, "sku": "MAP-A4" },
  { "name": "A3 в рамке", "price": 3200, "sku": "MAP-A3" }
]
```

---

# КАК ДОБАВЛЯТЬ ЗОНЫ ДОСТАВКИ

Зайдите в админку → `Доставка` → `Добавить зону`

В поле GeoJSON вставляете полигон.

Пример:

```json
{
  "type": "Polygon",
  "coordinates": [
    [
      [55.05, 51.76],
      [55.10, 51.76],
      [55.10, 51.80],
      [55.05, 51.80],
      [55.05, 51.76]
    ]
  ]
}
```

Потом ниже добавляете правила:
- зона 1, от 0, доставка 150
- зона 1, от 700, доставка 100
- зона 1, от 1500, доставка 0

# ЧТО НУЖНО ДОРАБОТАТЬ ПОД ВАШ ПРОЕКТ В ПЕРВУЮ ОЧЕРЕДЬ

Это уже не обязательно для запуска, но желательно дальше:

## 1. Яндекс.Карты
Сейчас форма заказа отправляет адрес текстом.
Чтобы доставка считалась по адресу автоматически, нужно:
- подключить Яндекс Maps JS API
- геокодировать адрес
- писать `lat/lng` в скрытые поля формы

## 2. R2 для загрузки фото
Сейчас фото — это URL.
Позже можно сделать загрузку файлов прямо из админки в Cloudflare R2.

## 3. Cloudflare Access
Лучше закрыть:
- `/admin/*`
- `/call-center/*`
- `/api/admin/*`
